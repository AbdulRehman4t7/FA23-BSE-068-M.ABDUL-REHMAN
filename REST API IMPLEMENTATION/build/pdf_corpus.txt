Designing REST API
Lecture Notes
Topic: Designing REST API Using Case Study
Duration: 2 Hours
Lecture Type: Theory + Practical (Step-by-Step)
Level: Computer Science Students
 Introduction to REST API (0 – 20 Minutes)
What is an API?
API (Application Programming Interface) is a set of rules that allows different software systems to
communicate with each other.
Real-life Example:
When you use Foodpanda or Careem, the mobile app communicates with a server using APIs.
The app sends a request:
“Show available restaurants”
The server responds with restaurant data in JSON format.
What is REST?
REST (Representational State Transfer) is an architectural style for designing networked applications.
It uses:
HTTP protocol
Standard HTTP methods
Stateless communication
REST Principles:

1. Stateless
2. Client-Server Architecture
3. Uniform Interface
4. Resource-based
HTTP Methods Used in REST
MethodPurposeReal-life Example
GETRetrieve dataView product
POSTCreate dataRegister user
PUTUpdate dataUpdate profile
DELETEDelete dataDelete account

Step 3: Create Books API

GET All Books
let books = [];
app.get('/books', (req, res) => {
    res.json(books);
});
POST Book
app.post('/books', (req, res) => {
    const book = req.body;
    books.push(book);
    res.status(201).json(book);
});
GET Book by ID
app.get('/books/:id', (req, res) => {
    const id = parseInt(req.params.id);
    const book = books.find(b => b.id === id);
    if (!book) {
        return res.status(404).json({message: "Book not found"});
    }
    res.json(book);
});
PUT Update Book
app.put('/books/:id', (req, res) => {
    const id = parseInt(req.params.id);
    const index = books.findIndex(b => b.id === id);
    if (index === -1) {
        return res.status(404).json({message: "Book not found"});
    }
    books[index] = req.body;
    res.json(books[index]);
});
DELETE Book
app.delete('/books/:id', (req, res) => {
    const id = parseInt(req.params.id);
    books = books.filter(b => b.id !== id);
    res.json({message: "Book deleted"});
});

Part 1: Introduction to API & REST (0 – 20 min)
What is an API?
API stands for Application Programming Interface.
 Real-life Example:
Think of a restaurant waiter:
You (client) give order.
Waiter (API) sends order to kitchen (server).
Kitchen prepares food.
Waiter brings food back to you.
So:
Client = Frontend
Server = Backend
API = Communication bridge
What is REST?
REST stands for Representational State Transfer.
It is an architectural style for building web services.
It uses HTTP protocol.
Part 2: REST Principles (20 – 40 min)
REST is based on 6 main principles:
 Client-Server Architecture
Frontend and backend are separate.
 Stateless
Each request must contain all necessary information.
Server does NOT store client session.
Example:
Every time you go to ATM, you insert card again.
 Cacheable
Responses can be cached to improve performance.

 Uniform Interface
Standard URL and HTTP methods.
 Layered System
Can have security layer, database layer etc.

GET Student By ID
app.get('/api/students/:id', (req, res) => {
    const student = students.find(s => s.id == req.params.id);
    if (!student) {
        return res.status(404).json({ message: "Student not found" });
    }
    res.json(student);
});
POST Create Student
app.post('/api/students', (req, res) => {
    const newStudent = {
        id: students.length + 1,
        ...req.body

    students.push(newStudent);
    res.status(201).json(newStudent);
});

PUT Update Student
app.put('/api/students/:id', (req, res) => {
    const student = students.find(s => s.id == req.params.id);
    if (!student) {
        return res.status(404).json({ message: "Student not found" });

    student.email = req.body.email;
    student.department = req.body.department;
    res.json(student);
});

app.delete('/api/students/:id', (req, res) => {
    students = students.filter(s => s.id != req.params.id);
    res.json({ message: "Student deleted successfully" });
});

REST API Security Overview
 Lecture Notes
Topic: REST API Security – CORS, XSS, DDoS | Authentication &
Authorization | Session Handling in REST
Duration: 2 Hours
Lecture Type: Theory
Designed for: Computer Science Students
 Lecture Plan Overview (2 Hours)
TimeTopic
0–30 minIntroduction to REST API Security + CORS

90–120 minSession Handling in REST + Wrap-up Activity
⏱ 0–30 Minutes
 Introduction to REST API Security
 What is REST?
REST (Representational State Transfer) is an architectural style for designing web services. REST APIs use
HTTP methods:
GET
POST
PUT

DELETE
Real-Life Example:
Think of a restaurant:
Menu = API endpoints
Order = Request
Food served = Response
Waiter = Server
 Why Security is Important?
If APIs are not secured:
Hackers can steal user data
Attackers can overload servers
Unauthorized users can access restricted data
Example:
Imagine a banking API without authentication — anyone could check or transfer money!
  CORS (Cross-Origin Resource Sharing)
Definition:
CORS is a security mechanism that controls how resources on a web server can be requested from
another domain.
Browsers enforce CORS.
Real-Life Example:
You live in Hostel A. Only students from Hostel A can enter.
If someone from Hostel B wants to enter, permission is required.
易 Same-Origin Policy
By default, browsers block:
Frontend: http://example.com
API: http://api.example.com
Even though related, they are different origins.
⚙ How CORS Works

Server sends headers like:
Access-Control-Allow-Origin: https://example.com
Access-Control-Allow-Methods: GET, POST
Access-Control-Allow-Headers: Content-Type
 CORS Security Best Practices
❌ Never use Access-Control-Allow-Origin: * for sensitive APIs
✅ Allow only trusted domains
✅ Restrict methods
✅ Validate tokens server-side
 Student Activity (5 Minutes)
Ask students:

⏱ 30–60 Minutes
 XSS (Cross-Site Scripting)
Definition:
XSS is an attack where malicious scripts are injected into trusted websites.
 Types of XSS
1. Stored XSS
2. Reflected XSS
3. DOM-based XSS
易 Real-Life Example:
A comment box allows users to post comments.
Attacker enters:
<script>alert("Hacked!")</script>
If not sanitized, every user visiting the page sees it.

 Impact of XSS
Cookie theft
Session hijacking
Redirect to malicious sites
 Prevention Techniques
Input validation
Output encoding
Use HTTPOnly cookies
Use Content Security Policy (CSP)
  DDoS (Distributed Denial of Service)
Definition:
DDoS is an attack where multiple systems flood a server with traffic to make it unavailable.
Real-Life Example:
Imagine 10,000 fake customers enter a shop and don’t buy anything — real customers cannot enter.
Famous Example:
In 2016, the company Dyn was attacked, affecting websites like:
Twitter
Netflix
GitHub
(This is a real-world referenced event.)
 DDoS Protection
Rate limiting
CAPTCHA
API Gateway
Load balancers
CDN services

 Student Quick Quiz (5 Minutes)

⏱ 60–90 Minutes
 Authentication vs Authorization
 Authentication (AuthN)
Definition:
Verifying who you are.
Example:
Username & password
OTP
Biometrics
 Authorization (AuthZ)
Definition:
Verifying what you can access.
Example:
Student cannot access admin panel
Teacher can upload marks
易 Real-Life Example
At University:
ID Card Check = Authentication
Access to Lab = Authorization
 Common Authentication Methods in REST
1. Basic Authentication
2. API Keys

3. Token-Based Authentication
4. JWT (JSON Web Token)
5. OAuth 2.0
 JWT (JSON Web Token)
Definition:
JWT is a compact, URL-safe token format used for authentication.
Structure:
Header.Payload.Signature
Example Use Case:
User logs in → Server generates JWT → Client stores token → Sends token in header:
Authorization: Bearer <token>
 OAuth 2.0
Used by:
Google
Facebook
Microsoft
Example:
"Login with Google"
 Class Discussion (5 Minutes)
Ask:
Why is JWT preferred in REST?

Is authentication enough without authorization?
⏱ 90–120 Minutes
 Session Handling in REST
❓ What is a Session?
A session stores user state between requests.

 Problem:
REST is stateless.
Each request must contain all information.
 Two Approaches to Session Handling
 Traditional Session-Based
Server stores session
Client stores session ID in cookie
Problem:
Not scalable
Hard for microservices
 Token-Based (Recommended for REST)
Server issues JWT
Client stores token
Each request includes token
Advantages:
Scalable
Stateless
Works with mobile apps
 Secure Session Practices
Use HTTPS
Set HttpOnly & Secure cookies
Short token expiry
Refresh tokens
Revoke compromised tokens
易 Complete Flow Example
1. User logs in
2. Server validates credentials

3. Server generates JWT
4. Client stores JWT
5. Client sends token in header
6. Server verifies token
7. Access granted or denied

 Key Takeaways
CORS protects browsers from cross-origin attacks
XSS targets users
DDoS targets availability
Authentication ≠ Authorization

 End of Lecture Summary
Security in REST APIs is not optional — it is essential.
Without:
Proper CORS → Browser misuse
XSS protection → User data theft
DDoS protection → Service unavailability
AuthN & AuthZ → Unauthorized access
Secure sessions → Account hijacking
A secure API is like a well-guarded university:
Gatekeeper (Authentication)
Access control (Authorization)
CCTV & Rules (Security policies)
Visitor log (Session handling)
